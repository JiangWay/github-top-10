## 每日一字

### 透視 (X-Ray)

「透視」——今日 10 檔的共同母題，也是 2025–2026 LLM 工程社群與資料社群同步跨進的共同方向：**把原本黑盒的東西，變成可以一眼看透的可觀測對象**。今日榜單裡：

- [langfuse/langfuse](https://github.com/langfuse/langfuse)（LLM 基礎建設）把每一次 LLM 呼叫、每一個 agent 決策、每一段 prompt 模板變成可追蹤的 trace，對標 LangSmith 的**LLM observability** 敘事。
- [open-metadata/OpenMetadata](https://github.com/open-metadata/OpenMetadata)（企業治理）把 Snowflake / BigQuery / Airflow / dbt / Superset 的資料血緣與 schema 變化畫成可瀏覽圖，這是**data observability** 的基礎設施。
- [zilliztech/claude-context](https://github.com/zilliztech/claude-context)（MCP 協定）以 BM25 + dense vector + AST 切塊把整份 repo 變成 Claude Code 看得見的語意上下文，官方敘事是「讓 coding agent 透視 codebase 而非盲搜」。
- [KeygraphHQ/shannon](https://github.com/KeygraphHQ/shannon) 與 [Z4nzu/hackingtool](https://github.com/Z4nzu/hackingtool)（資安）分別從「AI 全自動白盒 pentester」與「185+ 工具合集」兩個極端，提供**把網站 / API 漏洞可視化**的能力。
- [koala73/worldmonitor](https://github.com/koala73/worldmonitor) 與 [sansan0/TrendRadar](https://github.com/sansan0/TrendRadar)（情報監測）分別從英語 OSINT 地球與中文平台熱榜兩端，把**全球資訊暗流透視為視覺 dashboard**。
- [HKUDS/RAG-Anything](https://github.com/HKUDS/RAG-Anything)（RAG 框架）把 PDF / Office / 圖表 / 公式透過 MinerU + docling + VLM 拆成結構化知識圖譜，是**document observability** 的代名詞。
- [ruvnet/RuView](https://github.com/ruvnet/RuView)（硬體）用 ESP32 抓 WiFi CSI 還原人體姿態——字面意義上的「透視牆壁」。
- [Fincept-Corporation/FinceptTerminal](https://github.com/Fincept-Corporation/FinceptTerminal)（金融科技）把 Bloomberg Terminal 的 19,000+ 商品即時行情與 CFA 級分析帶到桌面——**把機構級金融資訊透視給散戶**。

連續 6 天敘事：04-18「進化」→ 04-19「主權」→ 04-20「平替」→ 04-21「邊界」→ 04-22「脈絡」→ 04-23「透視」。從「自己訓練 → 自己擁資料 → 自己選工具 → 自己定規則 → 自己組脈絡 → 自己看清內部」——今日「透視」是前 5 天命題收束後的**自然產物**：當 AI 已經擁有能力、主權、替代方案、邊界與脈絡，**下一步必然是讓黑盒變可觀測**，才能進入工程化維運的實用階段。
