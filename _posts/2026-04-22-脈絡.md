## 每日一字

### 脈絡 (Context)

「脈絡」——今日榜單最密集的共通詞。2025–2026 LLM 社群幾乎所有熱門專案都在同一場戰爭：**擴大、結構化、即時化 AI 能看到的 context**。今日 5 檔首次上榜的 repo，有 3 檔正面對這個命題：

- [zilliztech/claude-context](https://github.com/zilliztech/claude-context)（MCP 協定）用 BM25 + dense vector + AST chunk + Merkle tree 增量索引把**整份 codebase 塞進 Claude Code 的工作 context**——官方文案稱可省 40% token；核心挑戰是 code 變動時如何快速更新向量索引。
- [HKUDS/RAG-Anything](https://github.com/HKUDS/RAG-Anything)（RAG 框架）把 PDF / Office / 圖片 / 表格 / 數學式以 MinerU + Docling + PaddleOCR 解析後，透過 5 階段 pipeline 生成**雙多模態知識圖譜**，解決純文字 RAG 在企業文件 / 學術 PDF 場景的資訊損失問題。
- [sansan0/TrendRadar](https://github.com/sansan0/TrendRadar)（情報監測）爬取 30+ 中文熱榜平台，以關鍵字 + AI 過濾聚合成個人化熱榜，再推播到 WeChat / Feishu / Telegram——本質是把**跨平台網路輿情的脈絡**蒸餾成個人 dashboard。

加上 [microsoft/ai-agents-for-beginners](https://github.com/microsoft/ai-agents-for-beginners) 的 12+ 課教材正面教授「context engineering / memory / MCP 協定」——「脈絡」成為今日教學與工具並軌的共通命題。

接續 04-18「進化」→ 04-19「主權」→ 04-20「平替」→ 04-21「邊界」——今日「脈絡」推進的是**「AI 真正需要看到什麼才會有用」**這個命題：從「自己訓練（進化）→ 自己擁有資料（主權）→ 自己選工具（平替）→ 自己定規則（邊界）→ 自己組織 AI 看得到的資訊（脈絡）」——五天下來的敘事走入工程層面的實作細節。
