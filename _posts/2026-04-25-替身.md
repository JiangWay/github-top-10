## 每日一字

### 替身 (Stand-in)

「替身」——今日 10 檔的共同身分，也是 2026 年 4 月下旬開源生態最尖銳的訊號：**社群正在一檔一檔地把商用產品「收編」為開源替身**。「替身」與 04-20 的「平替」相關但不同——「平替」是「找便宜版本」（買不起 Cursor 就用 Cline），「替身」是「找開源代演者」（不買 Bitwarden cloud、自架一個 Rust 重寫的同款服務）。差別在於「替身」更徹底：不只是商業上的替代，更是**身份上的扮演**——同樣的 API、同樣的 client、同樣的功能集，只是換成你能 fork、能審 code、能自架的版本。

今日替身名冊：

- [dani-garcia/vaultwarden](https://github.com/dani-garcia/vaultwarden)（資安）是最教科書的替身——2018 年起以 Rust 完整實作 Bitwarden Server API，沿用 Bitwarden 官方 mobile / browser client 即可連線。2021 年原名 `bitwarden_rs` 應 Bitwarden 商標要求改名 `vaultwarden`，但 Bitwarden 也同意以「powered by Vaultwarden」標註相容存在——這是替身與被替身者「和平共處」的標準範本。
- [PostHog/posthog](https://github.com/PostHog/posthog)（開發者工具）是 Mixpanel / Amplitude 的替身——YC W20 老資格、33k stars，自架完整 product analytics + session replay + feature flags + error tracking 棧。其商業策略也是「替身」邏輯：核心 MIT、企業功能在 `/ee` 目錄、外加完全 FOSS 鏡像 [PostHog/posthog-foss](https://github.com/PostHog/posthog-foss)。
- [open-metadata/OpenMetadata](https://github.com/open-metadata/OpenMetadata)（企業治理）是 Atlan / Collibra 的替身——data catalog + lineage + MCP server，連續 3 天上榜後仍然有 +530 stars 的續航。
- [Alishahryar1/free-claude-code](https://github.com/Alishahryar1/free-claude-code)（LLM 客戶端）是 Claude Code 訂閱的替身——FastAPI 協定轉譯，把 Anthropic 端點轉到 NVIDIA NIM / OpenRouter / DeepSeek。今日衝上絕對榜 #1，stars 從 5,166 → 8,191 一日內再加 3,025。
- [huggingface/ml-intern](https://github.com/huggingface/ml-intern)（AI Agent 框架）是 Cursor / Devin 在 ML 工程領域的替身。本日 +2,981 stars 的 5.6× 加速是本站見過最強烈的「次日爆發」訊號之一。
- [zilliztech/claude-context](https://github.com/zilliztech/claude-context)（MCP 協定）是 GitHub Copilot enterprise code search 的替身——把整個 codebase 變成 agent 的語意索引。
- [google/osv-scanner](https://github.com/google/osv-scanner)（資安）是 Snyk / Dependabot 的替身——Google 官方 CLI、Apache-2.0、背靠 [osv.dev](https://osv.dev) 公開漏洞資料庫，11+ 語言生態系、19+ lockfile 類型；2025 年 V2 milestone 引入 layer-aware 容器掃描與引導式修復。
- [Z4nzu/hackingtool](https://github.com/Z4nzu/hackingtool)（資安）是商用 pen test toolkit（Cobalt Strike、Burp Suite Professional 等）的「替身合輯」——Python 選單包 185+ 工具的 install + run。
- [codecrafters-io/build-your-own-x](https://github.com/codecrafters-io/build-your-own-x)（教學資源）是「替身的元教材」——教你親手 build Redis、Git、Docker、Bittorrent、瀏覽器、shell 的最小可行替身。GitHub 全球 stars #1，背後是 YC S22 的 [codecrafters.io](https://codecrafters.io) 互動式 coding 挑戰平台。

唯一的**假替身**是 [Anil-matcha/Open-Generative-AI](https://github.com/Anil-matcha/Open-Generative-AI)（語音與多媒體）——自稱 200+ 開源模型 image/video gen 平台，實際 99% 走 `api.muapi.ai` 付費 SaaS（只有 v1.0.3 macOS Metal binary 透過 [leejet/stable-diffusion.cpp](https://github.com/leejet/stable-diffusion.cpp) 做真本地推理）；README 寫 MIT 但 repo 沒 LICENSE 檔、五個 release 壓在 35 天內、hosted 版直接掛在 muapi.ai 子網域、與 muapi.ai 的關係未揭露——疑似 muapi.ai 自家行銷前端。今日替身浪潮裡，這檔正好示範了「假替身」的反例：**披著開源外衣的閉源前端**。

連續 8 天敘事：04-18「進化」→ 04-19「主權」→ 04-20「平替」→ 04-21「邊界」→ 04-22「脈絡」→ 04-23「透視」→ 04-24「代勞」→ 04-25「替身」。從「自己訓練 → 自己擁資料 → 自己選工具 → 自己定規則 → 自己組脈絡 → 自己看清內部 → 讓 AI 替自己做事 → **每件商用產品都有一個開源版本可以扮演同樣的角色**」——前 7 天的命題是「能力」，今日的命題是「替代」。當「代勞」的對象一一被收編，世界就有了一張完整的開源替身名冊。
