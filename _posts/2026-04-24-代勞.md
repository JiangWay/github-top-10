## 每日一字

### 代勞 (Delegation)

「代勞」——今日 10 檔的共同母題，也是 2026 年 4 月下旬 LLM agent 社群最鮮明的轉向：**從「讓 AI 看清楚」（透視）進到「讓 AI 替我做」（代勞）**。今日榜單裡：

- [huggingface/ml-intern](https://github.com/huggingface/ml-intern)（AI Agent 框架）把 ML 工程師工作**整包外包**——「reads papers, trains models, and ships ML models」。Hugging Face 官方把 Claude Code、Codex 的定位複製到 ML 領域，是 2026 年「垂直 agent」浪潮的旗艦案例。
- [cline/cline](https://github.com/cline/cline)（LLM 客戶端）是最直接的「coding 代勞工」——VS Code 打開就有一個會自己改檔案、自己執行指令、自己開瀏覽器的 agent。
- [Alishahryar1/free-claude-code](https://github.com/Alishahryar1/free-claude-code)（LLM 客戶端）則是一種**代勞的代勞**——連「付 Claude Code 訂閱」這件事都被外包掉：架設 FastAPI 協定轉譯層，把 Claude Code 的 Anthropic 呼叫改打 NVIDIA NIM（每分鐘 40 次免費額度）、OpenRouter、DeepSeek、LM Studio。技術上是合法 BYOK 代理，倫理上踩商標灰色（沿用 Claude Code 品牌卻不付 Anthropic 一分錢）；46.23% 的單日 growth rate 說明社群對「Claude Code 繞路免費用」的需求強烈。
- [zilliztech/claude-context](https://github.com/zilliztech/claude-context)（MCP 協定）扮演**代勞基礎設施**，把 codebase 灌成 agent 能直接讀的語意索引，免除 agent 每次 grep+讀檔的 token 消耗。
- [HKUDS/RAG-Anything](https://github.com/HKUDS/RAG-Anything)（RAG 框架）與 [open-metadata/OpenMetadata](https://github.com/open-metadata/OpenMetadata)（企業治理）分別把 **PDF 整理** 與 **資料血緣維護**——兩件最耗人類心力的重複勞動——交給自動化 pipeline。
- [microsoft/ai-agents-for-beginners](https://github.com/microsoft/ai-agents-for-beginners)（教學資源）則是「教人類如何製造代勞」的 12 課入門系列，象徵這股風潮已滲透到一般工程師的學習優先序。
- [Z4nzu/hackingtool](https://github.com/Z4nzu/hackingtool)（資安）是**前 AI 時代**的代勞典範：把 185+ 工具的 `git clone + install + run` 步驟整包收進 Python 選單；雖然不含 agent，與今日主旋律精神一致。
- [ruvnet/RuView](https://github.com/ruvnet/RuView)（硬體）與 [PowerShell/PowerShell](https://github.com/PowerShell/PowerShell)（開發者工具）是今日兩條離題線——前者 WiFi CSI 硬體 agent 進入衰退期（連 5 天），後者因 2026-04-21 的 v7.6.1 servicing update 短暫上榜。

連續 7 天敘事：04-18「進化」→ 04-19「主權」→ 04-20「平替」→ 04-21「邊界」→ 04-22「脈絡」→ 04-23「透視」→ 04-24「代勞」。從「自己訓練 → 自己擁資料 → 自己選工具 → 自己定規則 → 自己組脈絡 → 自己看清內部 → **讓 AI 替自己做事**」——今日「代勞」是前 6 天命題的**自然收束**：當你已經能進化、擁主權、有平替、知邊界、組脈絡、看透內部，**最終目的是把這些能力託付給 agent 去執行**。
