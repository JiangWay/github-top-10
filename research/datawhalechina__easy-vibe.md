# [datawhalechina/easy-vibe](https://github.com/datawhalechina/easy-vibe) — 深度研究

## 深度研究（2026-05-10 首次）

### 專案定位
[datawhalechina/easy-vibe](https://github.com/datawhalechina/easy-vibe) 是中國開源 ML 社群 Datawhale 在 2025-12 末推出的「Vibe Coding 2026」課程。主打用自然語言描述需求、由 AI 協作生成可上線產品的「會說話就會編程」教學路徑，定位為零基礎使用者的第一門現代程式設計課。

### 核心架構 / 主要概念
課程分四階：
- **Stage 0** — 用小遊戲建立 AI 直覺
- **Stage 1** — 從產品經理視角做需求發掘到原型
- **Stage 2** — 補上前後端、Git/DB、雲端部署、Stripe 金流，產出 SaaS 級 capstone
- **Stage 3** — 深入 Claude Code 生態（MCP、Skills、Agent Teams）、跨平台交付（小程式 / iOS / Android / PWA / 桌面）與進階 RAG

教材以 Next.js 寫成互動站點（JavaScript 為主要語言），含模擬鍵鼠導引、動畫化 AI 原理、可視化 terminal。

### 目標使用者
零基礎初學者、想快速驗證點子的 PM 與 founder、AI 時代學生、從原型走向全棧的初級開發者，以及想升級 AI 協作流程的中高階工程師。

### 與類似專案的差異
比同隊的 [datawhalechina/vibe-vibe](https://github.com/datawhalechina/vibe-vibe) 更聚焦零基礎入門與互動視覺化教材；相對於 [freeCodeCamp/freeCodeCamp](https://github.com/freeCodeCamp/freeCodeCamp) 之類語法導向課程，easy-vibe 把 AI IDE（Cursor、Trae、Claude Code）視為開發夥伴，以「對話描述產品 → 迭代修正」取代背語法。涵蓋面從前端到金流、行動端到 RAG，scope 比一般 vibe coding 教程更完整。

### 外部評論
- Vibe Sparking AI 形容它是「Zero to Hero, Project-Based Learning」，強調專案導向而非語法導向 [(來源)](https://www.vibesparking.com/en/blog/ai/2026-01-06-datawhale-easy-vibe-vibe-coding-course/)
- 搜狐 task1 學習筆記反映社群已有 task 制共學在進行，呼應 Datawhale 慣用的開源共學模式 [(來源)](https://www.sohu.com/a/986026278_121124374)
- 概念源頭可追溯到 Andrej Karpathy 於 2025-02 提出的 vibe coding 一詞，課程定位緊扣此潮流。
