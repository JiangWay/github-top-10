# [Donchitos/Claude-Code-Game-Studios](https://github.com/Donchitos/Claude-Code-Game-Studios)

## 深度研究（2026-04-18 首次）

### 專案定位
[Donchitos/Claude-Code-Game-Studios](https://github.com/Donchitos/Claude-Code-Game-Studios) 將單一 Claude Code 會話改造成完整的「AI 遊戲開發工作室」。口號為 "49 AI agents, 72 workflow skills, and a complete coordination system mirroring real studio hierarchy"，訴求從概念到發佈的結構化流程，而非放任式自動生成。

### 核心架構 / 主要概念
三層階級制度：
- 第一層「Directors」：創意、技術、製作導演，守護願景與品質閘。
- 第二層「Department Leads」：設計、程式、美術等部門主管。
- 第三層「Specialists」：49 位領域專家執行實作。

內建 72 個 slash 指令技能、12 個自動化 hooks（資產、提交、技能測試驗證）、11 項程式碼標準強制。互動協議為 Question → Options → Decision → Draft → Approval，並可切換 full / lean / solo 審查強度。

### 目標使用者
獨立開發者與小型團隊，需要 AI 協作但保留人工決策；支援 Godot 4、Unity、Unreal Engine 5 三大引擎。

### 與類似專案的差異
相較於單純角色扮演 prompt 包或 [BMad-Code/BMAD-METHOD](https://github.com/BMad-Code/BMAD-METHOD) 類通用代理框架，此專案專精遊戲開發全生命週期，並將「製作管理、品質閘、ADR 架構決策紀錄」納入工作流，強調結構化而非自駕模式。
